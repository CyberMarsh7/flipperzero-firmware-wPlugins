◆ usb_can_view_alloc()

usbCanView * usb_can_view_alloc ( )

This function allocate view usbCanView::view element and register usb_can_draw_callback and usb_can_input_callback callbacks

This function is called on application start when memory is allocated through usb_can_app_alloc(). It perform following actions:

  1. allocate memory space for usbCanView object via malloc
  2. initialize usbCanView:view pointer allocating dedicated memory space via malloc
  3. allocate memory model (updated via MODEL getters) holding data displayed to the user on flipper screen( UsbCanModel)
  4. set pointer to be passed to usbCanView:view object callbacks through view_set_context(). Here this is the usbCanView object created holding all needed references to operate the VIEW component.
  5. register draw callback to furi (Flipper Universal Registry Implementation) gui stack through view_set_draw_callback
  6. register user input callback to furi gui stack via view_set_input_callback (when a flliper key is pressed)
    Returns
    a pointer to an allocated and initialized usbCanView object.
