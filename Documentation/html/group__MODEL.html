◆ usb_can_disable()

void usb_can_disable ( UsbCanBridge *  usb_can)

function called to leave application operationnal mode (free memory).

this function is called when back button is pressed by the user through usb_can_scene_usb_can_on_exit(). It performs the following actions:

  1. sends killing signal to application main thread ( usb_can_worker())
  2. wait thread terminates
  3. free memory (and ressources) related to application control logic allocated in usb_can_disable()
    Parameters
    [in,out]usb_canmemory structure holding application (USB and CAN) control logic to free.

◆ usb_can_enable()

UsbCanBridge * usb_can_enable ( UsbCanApp *  app,
usbCanState  state 
)

Entry point of application's operating mode (cf. usb_can_worker()) called when one of the can-fd-hs application menu item (USB-UART-Bridge, USB loopback, CAN test) is choosen.

This function is called via usb_can_scene_usb_can_on_enter() when one of the flipper zero can-fd-hs applications mode is chosen by the user. It will:

  1. allocate memory to store USB and CAN control logic internal data app->usb_can_bridge ( UsbCanBridge)
  2. set application internal state ( app.usb_can_bridge.state) according to application mode chosen by the user
  3. start application main thread : usb_can_worker().
    Parameters
    [in,out]appstructure holding application related data (model, view, control logic)
    [in]stateapplication requested state. It correspond to application mode (menu item) chosen by the user.
